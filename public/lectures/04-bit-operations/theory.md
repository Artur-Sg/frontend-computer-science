# Лекция 04 — Операции над битами

## Зачем вообще биты

Процессор работает с байтами, но иногда это расточительно. Если хранить флаги или булевы значения в виде отдельных байтов, мы теряем память. Побитовые операции позволяют упаковывать данные и извлекать нужные биты.

## Бинарные форматы и заголовки

Во многих бинарных форматах есть маленькие заголовки и метаданные (версия, флаги, хеш). Их удобно упаковывать в несколько бит внутри одного байта.

## Побитовые операторы в JS

JavaScript приводит `number` к 32‑битному целому и выполняет операцию побитово.

- `&` — побитовое «И» (1, если оба бита 1)
- `|` — побитовое «ИЛИ» (1, если хотя бы один бит 1)
- `^` — исключающее ИЛИ (1, если биты разные)
- `~` — побитовое НЕ
- `<<` — сдвиг влево (заполнение нулями справа)
- `>>` — арифметический сдвиг вправо (сохраняет знак)
- `>>>` — логический сдвиг вправо (заполняет нулями слева)

Важно: любая побитовая операция превращает число в 32‑битное.

## Маски

**Маска** — число, в котором единицы стоят на нужных позициях.

```js
function createMask(position, size) {
  let mask = 1;
  mask = (mask << size) - 1; // размер маски (например, 8 единиц)
  mask = mask << position;   // сдвигаем на нужную позицию
  return mask;
}
```

Маски используют для чтения/записи отдельных битов и компактного хранения признаков.

## Сдвиги

### `<<` (влево)

Сдвигает биты влево, справа дописывает нули. Это эквивалентно умножению на 2^k.

```js
0b0101 << 2 // 0b010100 (5 * 4 = 20)
```

### `>>` (вправо, арифметический)

Сдвигает вправо, слева повторяет бит знака.

```js
0b0101 >> 2 // 0b0001 (5 / 4 = 1)
```

### `>>>` (вправо, логический)

Всегда дописывает нули слева, делает число беззнаковым.

```js
-1 >>> 0 // 4294967295
```

## Чтение и упаковка данных

Побитовые операции позволяют кодировать несколько значений внутри одного числа.

**Пример BCD внутри числа:**

```js
// чтение второй цифры справа (4 бита)
(bcd >>> 4) & 0b1111
```

**Запись:**

```js
9 << 8 | 7 << 4 | 1
```

## Битовые флаги

```js
const CAN_READ = 0b0001;
const CAN_WRITE = 0b0010;
const CAN_DELETE = 0b0100;

const editor = CAN_READ | CAN_WRITE;

if (editor & CAN_READ) {
  // доступ есть
}
```

Это экономичнее и быстрее, чем хранить объект из `true/false`.

## Оптимизация деления на степень двойки

Сдвиги — быстрый способ умножения/деления на 2^k. JIT‑движки часто оптимизируют `%` и `/` для степеней двойки.

## Побитовые особенности JS

- Все операции — в 32‑битной области.
- `2 ** 32 >>> 0 === 0` (переполнение)
- `>>>` возвращает беззнаковую интерпретацию

## Полезные примеры

### Проверка знака

```js
function isNegative(num) {
  return (num & (1 << 31)) !== 0;
}
```

### XOR‑сумматор

```js
function add(a, b) {
  while (b !== 0) {
    const sum = a ^ b;
    const carry = (a & b) << 1;
    a = sum;
    b = carry;
  }
  return a;
}
```

### Генератор XorShift

```js
class XorShift {
  constructor(seed = Date.now()) {
    this._seed = seed;
  }
  random() {
    this._seed ^= this._seed << 13;
    this._seed ^= this._seed >> 17;
    this._seed ^= this._seed << 5;
    return this._seed < 0
      ? 1.0 + this._seed / 0x80000000
      : this._seed / 0x7fffffff;
  }
}
```

## Итог

- Битовые операции — фундамент вычислений и оптимизаций.
- Маски — ключевой инструмент для упаковки данных.
- В JS всё происходит в 32‑битном Int32/Uint32.
