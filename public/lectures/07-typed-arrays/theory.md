# Лекция 07 — ArrayBuffer и типизированные массивы в JS

## Выжимка лекции

- `ArrayBuffer` — это «сырая» память (поток байт).
- `TypedArray` — представление (`view`) над этой памятью с фиксированным типом элемента.
- `DataView` — низкоуровневый API для чтения/записи разных типов по смещению в байтах.
- Один и тот же буфер можно читать разными view без копирования.
- Для бинарных протоколов критичны: выравнивание, смещения, порядок байт (`endianness`).

## TypedArray: что важно помнить

- Нет дырок и гетерогенности: данные плотные и предсказуемые.
- Тип элемента фиксирован (`Uint8Array`, `Int32Array`, `Float64Array`, ...).
- При записи/чтении значения конвертируются в контракт типа.
- Возможны переполнения, обрезка и потеря точности (например, в `Uint8Array` и `Float16Array`).

## ArrayBuffer и модель view

`ArrayBuffer` не знает ничего про типы. Типы появляются только в момент создания view:

```js
const buffer = new ArrayBuffer(16);
const u8 = new Uint8Array(buffer);
const i32 = new Int32Array(buffer);
```

`u8` и `i32` смотрят на один и тот же буфер, поэтому изменения видны в обоих представлениях.

## Смещение, длина и выравнивание

При создании view над частью буфера:

```js
new Uint16Array(buffer, byteOffset, length);
```

- `byteOffset` задаётся в байтах.
- Для некоторых типов нужен выровненный offset: например, для `Uint16Array` смещение должно быть кратно 2.
- Неверное выравнивание приводит к `RangeError`.

## Endianness и DataView

- У многобайтных чисел важен порядок байт: `Little-Endian` vs `Big-Endian`.
- Типизированные массивы обычно используют «родной» порядок платформы.
- `DataView` позволяет явно выбирать порядок при чтении/записи:

```js
view.getUint32(0, true); // little-endian
view.getUint32(0, false); // big-endian
```

## Строки и UTF-8

Строки для бинарных форматов обычно кодируют через UTF-8:

- `TextEncoder` -> `Uint8Array`
- `TextDecoder` <- `Uint8Array`

Это и есть базовый инструмент для сериализации строк в бинарный буфер.

## Почему это полезно на практике

- Работа с бинарными файлами и протоколами.
- Минимизация копирования при передаче данных.
- Более плотное хранение, контроль над памятью и форматом.
- Основа для собственных структур: кортежей, структур и tagged union поверх `ArrayBuffer`.
