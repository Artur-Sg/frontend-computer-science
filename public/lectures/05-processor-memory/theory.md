# Лекция 05 — Процессор и память

## Почему это важно

Процессор умеет считать очень быстро, а память — медленно. Из‑за этого реальная скорость программы часто определяется
не количеством операций, а тем, как данные лежат в памяти и как мы к ним обращаемся.

## Иерархия памяти

- Регистры — самые быстрые, очень маленькие.
- L1/L2/L3 кэши — быстрые, но ограниченные по объёму.
- RAM — большая, но медленная.
- Диск/SSD — ещё медленнее.

Процессор всегда пытается читать данные из кэша. Если данных нет, возникает кэш‑промах и чтение идёт из RAM.

## Кэш‑линии

Кэш загружает данные не по одному байту, а блоками фиксированного размера — **кэш‑линиями** (обычно 64 байта).
Если данные лежат рядом, они попадают в одну линию и читаются «пакетом».

**Вывод:** последовательный доступ по памяти быстрее, чем «прыжки».

## Row‑major vs Column‑major

Двумерные массивы в памяти хранятся построчно (row‑major). Поэтому:

- обход по строкам → последовательный доступ → быстрее;
- обход по столбцам → шаг через всю строку → медленнее.

## Массив значений vs массив ссылок

Обычный массив JS хранит **указатели**, а не сами значения. Объекты разбросаны по куче, поэтому кэш работает хуже.

Типизированные массивы (`Uint8Array`, `Float32Array`) хранят данные **плотно**, без указателей:

- меньше памяти;
- лучше локальность;
- меньше нагрузки на GC.

## Предсказание ветвлений

Процессор пытается угадать результат `if`. Если угадывает — код быстрый. Если нет — теряется производительность.
Случайные условия внутри горячих циклов — плохая идея.

## SIMD и векторизация

Современные процессоры умеют выполнять одну инструкцию сразу над несколькими числами (SIMD). JIT/компиляторы
пытаются автоматически векторизовать простые циклы. Помогают:

- типизированные массивы;
- отсутствие условий внутри цикла;
- последовательный доступ.

## Итоги

- Данные нужно хранить в том порядке, в котором их чаще всего обходят.
- Последовательный доступ может быть в разы быстрее случайного.
- Типизированные массивы дают лучшую локальность и меньше мусора.
- Ветвления внутри горячих циклов сильно влияют на скорость.
