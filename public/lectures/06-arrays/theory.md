# Лекция 06 — Фундаментальные структуры данных. Массивы

## Массив как базовая структура

Память можно представить как большой массив байт. Массив даёт доступ к элементу за **O(1)**, потому что адрес
вычисляется по формуле:

```
address = base + index * elementSize
```

Это делает массивы базой для большинства структур данных.

## Порядок и локальность

У элементов массива есть порядок. Последовательный обход эффективен для кэша процессора, случайный — хуже.

## JS массивы — «не совсем массивы»

В JavaScript обычные массивы могут хранить элементы разных типов. Поэтому под капотом это не «плотный» массив, а более
универсальная структура. Ближе всего к настоящим массивам — **TypedArray**.

## «Дырки» (holey arrays)

Если создавать массивы с пропусками, движок переходит на менее эффективное представление.

```js
const a = [];
a[100] = 42; // дырка -> падение производительности

const b = new Array(101).fill(0);
b[100] = 42; // плотный массив
```

**Вывод:** не создавайте массивы с дырками, если важна скорость.

## Предварительная длина

Если длина известна заранее, выгодно создавать массив фиксированного размера, а затем заполнять.

## Сложность операций

- Доступ по индексу — **O(1)**
- `push`/`pop` — **амортизированно O(1)**
- `shift`/`unshift` — **O(n)** (нужен сдвиг всех элементов)

## Ограничения размера

JS массивы ограничены длиной **2^32 - 1**. На практике лучше не приближаться к этим значениям из-за памяти.

## Итог

- Массивы — фундамент большинства структур данных.
- `push`/`pop` — быстрые, `shift`/`unshift` — медленные.
- Дырки в массивах сильно ухудшают производительность.
- TypedArray даёт более «плотное» представление данных.
