# Лекция 01 — Вычислители и кодирование информации

## Темы

- Вычислители и представление информации
- Системы счисления
- Бит и байт
- Данные, типы и структуры данных

## Цели

- Понять, как информация представляется в памяти
- Научиться переводить числа между системами счисления
- Понимать базовые понятия: бит, байт, слово, тип данных
- Отличать данные от их интерпретации

## Короткий конспект

- Программа — это поток битов. Один бит хранит 0 или 1.
- Минимальная порция данных, с которой обычно работает процессор, — байт (обычно 8 бит). Исторически размер байта мог отличаться, но в современных системах почти всегда 8 бит.
- Данные измеряются в байтах и их степенях. В памяти часто используется кратность 1024 (кибибайт, мебибайт).
- Кодирование — это отображение символов и значений в биты. Можно использовать фиксированную длину или переменную длину (быстрее или компактнее).
- Система счисления задает алфавит цифр и основание. Компьютер работает в двоичной системе, а для удобства чтения используются 8- и 16‑ричные записи.
- Числовые типы имеют фиксированный размер, поэтому переполнение приводит к отбрасыванию старших битов.
- Один и тот же набор битов может интерпретироваться по‑разному: как знаковое и беззнаковое число, как целое или как байты.
- Преобразование типа меняет значение, а приведение (реинтерпретация) — только способ чтения битов.

## Примеры

Запуск:

```bash
node lectures/01-encoding/examples.js
```

```js
const a = new Uint8Array([255]);
a[0] += 2;

console.log(a[0]); // 1
// 11111111
// +
// 0000 0010
// =
// (1) 0000 0001
```

### Перебор значений с ограничением по битам

```js
let a = 0;

for (let i = 0; i < 10; i++) {
  console.log(a);
  // ограничиваем число 3 битами
  a = (a + 1) & 0b111;
}
```

### Дополнение до двух (two's complement)

```js
console.log(-42 === ~42 + 1); // true
```

### Одинаковые биты — разная интерпретация

```js
const a = new Int8Array([-125]);
const b = new Uint8Array(a.buffer);

console.log(a[0]); // -125
console.log(b[0]); // 131
```

### Системы счисления и чтение чисел

```js
console.log((1255).toString(2));  // двоичная
console.log((1255).toString(8));  // восьмеричная
console.log((1255).toString(16)); // шестнадцатеричная
console.log((1255).toString(36)); // 0-9a-z

console.log(parseInt("10", 2));  // 2
console.log(parseInt("10", 8));  // 8
console.log(parseInt("10", 16)); // 16
console.log(parseInt("10", 36)); // 36
```

### Разные записи одного и того же числа

```js
console.log(0b1111_1111 === 255); // true
console.log(0xFF === 255);        // true
console.log(0xFF === 0o377);      // true
```

### RGB‑компоненты в разных системах

```js
console.log([0b1111_1111, 0b0000_0000, 0b0000_0000]); // [255, 0, 0]
console.log([0o377, 0o000, 0o000]);                   // [255, 0, 0]
console.log([0xFF, 0x00, 0x00]);                      // [255, 0, 0]
```

### Приведение vs преобразование

```js
const a = new Float64Array([42.1]);

// Приведение (реинтерпретация тех же бит)
const b = new Uint8Array(a.buffer);
console.log(b);

// Преобразование (новое значение)
const c = new Uint8Array(a);
console.log(c); // [42]
```
