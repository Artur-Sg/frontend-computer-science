# Лекция 03 — Кодирование чисел

## Иерархия числовых типов

- **Целые**: знаковые и беззнаковые.
- **Дробные**: плавающая точка (float), фиксированная точка (fixed‑point), рациональные.
- **Комплексные** — для научных вычислений.

Важно различать **двоичное** представление и **десятично‑двоичное** (BCD), когда нужна точность десятичной записи.

## Ключевые нюансы

1. **Разрядность** определяет диапазон и точность.
2. **Знаковость** отнимает бит под знак.
3. **Точность**: дроби в двоичной системе чаще всего приближённые.
4. **Переполнение**: разные языки ведут себя по‑разному (wrap/exception).

## BCD (Binary‑Coded Decimal)

Каждая десятичная цифра кодируется 4 битами:

```text
1234 → 0001 0010 0011 0100
```

### Варианты

- **Packed BCD**: 2 цифры в одном байте (старшая и младшая тетрады).
- **Unpacked BCD**: 1 цифра на байт (старшие биты — нули/служебные).

### Плюсы

- точное десятичное представление;
- простое преобразование в строку;
- удобно смотреть в памяти (0x12 0x34 → «1234»).

### Минусы

- расход памяти выше (~40%);
- арифметика сложнее и медленнее;
- редко поддерживается напрямую аппаратно/языками.

### Где используется

Финансы, бухгалтерия, POS‑терминалы, калькуляторы, COBOL, RTC‑часы.

### Отрицательные числа

- отдельный бит/байт под знак;
- десятичное дополнение (ten’s complement);
- зональный BCD (знак в старшей тетраде).

## Числа в JavaScript

### number (IEEE 754, 64‑бит double)

- безопасные целые: `-(2^53 - 1)` … `2^53 - 1`;
- есть `Infinity`, `-Infinity`, `NaN`;
- дроби часто неточны.

```js
console.log(Number.MAX_SAFE_INTEGER); // 9007199254740991
console.log(Number.isSafeInteger(9007199254740992)); // false

console.log(0.1 + 0.2); // 0.30000000000000004
```

### bigint

Целые числа произвольной длины.

```js
const big = 123456789012345678901234567890n;
const fromStr = BigInt("123456789012345678901234567890");
```

Нельзя смешивать `number` и `bigint` без явного преобразования.

## IEEE 754: структура double

```text
64 бита:
[ знак:1 ][ порядок:11 ][ мантисса:52 ]
```

Число вычисляется как:

```
(-1)^sign * (1 + mantissa / 2^52) * 2^(exponent - 1023)
```

### Почему 0.1 + 0.2 ≠ 0.3

Десятичные дроби не всегда имеют конечное двоичное представление:

```text
0.1₁₀ = 0.0001100110011...₂ (бесконечная периодическая)
```

Результат округляется → ошибка точности.

## Фиксированная точка

Фиксируем число бит под дробную часть (например, 16/16 в 32‑битном числе).

Плюсы: предсказуемая точность. Минусы: ограниченный диапазон, нужен заранее известный масштаб.

## NaN и ±0

```js
console.log(NaN === NaN); // false
console.log(Number.isNaN(NaN)); // true

console.log(0 === -0); // true
console.log(Object.is(0, -0)); // false
```

## Побитовые операции в JS

Побитовые операторы работают в **32‑битном знаковом** диапазоне:

```js
console.log(4294967295 >> 0);  // -1
console.log(4294967295 >>> 0); // 4294967295
console.log(4294967296 >> 0);  // 0
```

## Оптимизации в движках (SMI)

- **SMI**: маленькие целые числа хранятся прямо в указателе.
- **HeapNumber**: большие и дробные — в куче.

Это деталь реализации, но полезно для понимания производительности. В V8 есть `SMI` (Small Integer) и `HeapNumber`.

> Внутренний `%DebugPrint` работает только в Node с флагом `--allow-natives-syntax`.

## Типизированные массивы

Работа с бинарными данными через `ArrayBuffer` + `TypedArray`:

```js
const buffer = new ArrayBuffer(16);
const int32 = new Int32Array(buffer);
int32[0] = 42;

const uint16 = new Uint16Array(buffer);
console.log(uint16[0]); // 42 (младшие 2 байта)
```

### Преобразование при записи

```js
const arr = new Uint8Array(1);
arr[0] = 300; // 44
arr[0] = -1;  // 255
```

## Итоги

- BCD точен в десятичной системе, но дорог по памяти.
- `number` — IEEE 754, быстрый, но с ограниченной точностью.
- `bigint` — точность, но медленнее и без смешения с `number`.
- Побитовые операции → 32‑битная область.
- Typed Arrays — ключ к бинарным данным и производительности.
- отдельный байт/бит под знак;
- дополнение до 9/10 (ten’s complement);
- зональный BCD (знак в старшей тетраде).

### Разложение на разряды

BCD удобно разбирать на цифры — это просто чтение по 4 бита:

```js
function decomposeBCD(num, len) {
  for (let i = 0; i < len; i += 1) {
    const mask = 0b1111 << (i * 4);
    console.log((num & mask) >> (i * 4));
  }
}
```

Для обычного двоичного числа нужно деление на 10:

```js
function decompose(num) {
  do {
    console.log(num % 10);
    num = Math.floor(num / 10);
  } while (num > 0);
}
```

### Пример BCD‑класса

```js
class BCD {
  static LEN = 8;
  mem = 0;
  constructor(num) {
    let i = -1;
    do {
      this.mem |= (num % 10) << ((i += 1) * 4);
      num = Math.floor(num / 10);
    } while (num > 0);
  }
  toString() {
    let str = '';
    let prefix = '';
    for (let i = 0; i < BCD.LEN; i += 1) {
      const mask = 0b1111 << (i * 4);
      const s = String((this.mem & mask) >> (i * 4));
      prefix += s;
      if (s !== '0') {
        str = prefix + str;
        prefix = '';
      }
    }
    return str || '0';
  }
}
```

## Float64 в битах

```js
function float64toString(num) {
  const float64 = new Float64Array([num]);
  const bytes = new Uint8Array(float64.buffer);
  let str = '';
  for (let i = bytes.length - 1; i >= 0; i -= 1) {
    str += bytes[i].toString(2).padStart(8, '0');
    if (i !== 0) str += '_';
  }
  return str;
}
```

## Разная точность float

```js
console.log(new Float16Array([1.3])[0]); // 1.2998046875
console.log(new Float32Array([1.3])[0]); // 1.2999999523162842
```
